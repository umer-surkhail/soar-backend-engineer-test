SOAR API

Welcome. This service exposes a REST-style API for authentication, schools, classrooms, and students. All API requests use the base path /api/:moduleName/:fnName and return JSON.

Authentication

Obtain a long-lived token via login; send it on subsequent requests in the token header (or as configured).

  • POST /api/auth/login — Body: email, password. Returns longToken, userId, role, schoolId.
  • POST /api/auth/createUser — Superadmin only. Body: email, password, role (superadmin | school_admin), optional schoolId for school_admin.

Schools

School CRUD and listing (superadmin).

  • POST /api/school/create — Body: name, address, phone.
  • GET /api/school/get?id= — Get one school by id.
  • GET /api/school/list — List all schools.
  • POST /api/school/update — Body: id, optional name, address, phone, profile.
  • POST /api/school/remove — Body: id.
  • GET /api/school/getProfile?id= — Get school profile by id.

Classrooms

Classroom CRUD (school admin for their school).

  • POST /api/classroom/create — Body: schoolId, name, capacity, optional resources.
  • GET /api/classroom/get?id= — Get one classroom.
  • GET /api/classroom/list — List classrooms (query params as required).
  • POST /api/classroom/update — Body: id, optional fields.
  • POST /api/classroom/remove — Body: id.

Students

Student CRUD plus enroll and transfer.

  • POST /api/student/create — Create a student.
  • GET /api/student/get?id= — Get one student.
  • GET /api/student/list — List students.
  • POST /api/student/update — Update a student.
  • POST /api/student/remove — Remove a student.
  • POST /api/student/enroll — Enroll student in a classroom.
  • POST /api/student/transfer — Transfer student between classrooms.

Responses use { ok: true, data: ... } on success and { ok: false, message: ... } or { ok: false, errors: ... } on failure. Protected routes require a valid token header. Rate limiting applies to /api/.
